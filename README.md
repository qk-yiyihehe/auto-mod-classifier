# 自动筛选模组分类器 2.01

这是 `2.00` 之后补的一个小版本。

这次更新的重点很明确：`MC百科` 现在能处理带验证码的页面了，`CurseForge` 也接进来了，不过目前还是测试版。如果你主要靠 `MC百科` 做兜底筛选，那这一版比 `2.00` 更适合直接用。

如果你想要第一版对外发布时的老版本，请看 `v2.0.0`。当前仓库主线对应的是 `2.01`。

## 2.01 这次更新了什么

- `MC百科` 查询改走真实浏览器
- 遇到验证码或拦截页时，可以手动处理后继续
- 增加 `CurseForge` 兜底查询入口
- 保留 `2次筛选`，只回头重试首轮 `unknown`
- 继续保留 `一键制作服务端模式`

## 现在能做什么

- `Mod筛选模式`
  - 自动分类为 `服务端保留 / 纯客户端 / 待人工确认`
  - 支持 `Fabric / Quilt / Forge / NeoForge`
  - 本地能判断的先本地判断，本地判不准再查 `Modrinth`
  - 还不够时可以继续用 `MC百科` 兜底
  - `CurseForge` 也能查，但目前先当补充手段，不建议当主力
  - 会输出 `JSON / CSV / TXT` 报告，并单独标记损坏 Jar
- `一键制作服务端模式`
  - 选择整个客户端实例目录和新的空输出目录
  - 自动识别 `Minecraft` 版本、加载器类型和精确加载器版本
  - 支持 `Fabric / Forge / NeoForge` 自动建服
  - 自动下载官方安装器、复制可用模组、复制常见配置目录
  - 自动生成启动脚本，自动首启，写入 `eula=true`，再做一次启动验证

## 联网筛选这块，实际建议这样用

- 先让程序读本地元数据
- 本地判不准时再查 `Modrinth`
- 还想继续补判断时，用 `MC百科`
- `CurseForge` 先当测试入口，能帮上一部分，但我不建议把它当唯一结果来源

## 运行和打包

源码直接运行：

```powershell
pythonw .\自动筛选模组分类器.pyw
```

或者双击：

```text
启动自动筛选模组分类器.bat
```

重新打包就运行：

```text
打包发布.bat
```

脚本会在 `dist` 目录下生成两个文件：

- `自动筛选模组分类器.exe`
- `auto-mod-classifier-2.01.exe`

第二个文件名就是给 GitHub Release 准备的。

## 需要注意

- `MC百科 / CurseForge` 这两路现在都依赖真实浏览器，机器上最好有 `Chrome` 或 `Edge`
- 模组联网分类会受网络环境影响，速度不会完全稳定
- 一键建服不会修改客户端源目录
- 如果版本清单里识别不出精确加载器版本，程序会直接报错，不做模糊猜测
- `Quilt` 目前只保留模组筛选识别，不进入自动建服流程
