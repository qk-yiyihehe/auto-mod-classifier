# 自动筛选模组分类器 2.08

一个面向 Windows 的 Minecraft 模组筛选与自动建服工具。

仓库当前上传的是 `2.08` 版本源码和对应打包成品，重点是把整合包里的客户端模组筛出来，并尽量自动生成可启动的服务端。

## 主要功能

- `Mod 筛选模式`
  - 选择 `mods` 目录后，自动分类为 `服务端保留 / 纯客户端 / 待人工确认`
  - 支持 `Fabric / Quilt / Forge / NeoForge`
  - 支持可选 `2次筛选`，只重试首轮 `unknown`
- `一键制作服务端模式`
  - 选择整个客户端实例目录和新的空输出目录
  - 自动识别 `Minecraft 版本 + 加载器类型 + 精确加载器版本`
  - 自动下载官方服务端安装器
  - 自动筛选并复制模组
  - 自动复制常见配置目录
  - 自动生成启动脚本，并执行两次启动验证

## 2.08 版本重点

- MC百科 请求增加同 key 并发去重，减少重复请求
- `2次筛选` 只重试 `unknown`，并使用新的分类器实例重新联网
- `lib / library / api / mod` 后缀宽松匹配继续保留
- 一键建服失败时也会尽量保留模组筛选报告、目录摘要和日志
- 多版本候选改为自然排序
- Forge / NeoForge 的 `win_args.txt` 按标准路径优先定位

## 直接运行源码

需要本机已安装 Python 3。

```powershell
pythonw .\自动筛选模组分类器.pyw
```

或者直接双击：

```text
启动自动筛选模组分类器.bat
```

## 重新打包

仓库内已提供：

```text
打包发布.bat
```

它会调用 PyInstaller 生成带图标的单文件 exe。

## 使用说明

### Mod 筛选模式

- 输入一个 `mods` 目录
- 自动读取 jar 元数据
- 本地先判断，判不准再查 `Modrinth`，必要时再查 `MC百科`
- 最终输出分类报告，并把 `纯客户端` 与 `待人工确认` 分目录整理

### 一键制作服务端模式

- 输入整个客户端实例目录，不是单独的 `mods` 目录
- 也支持直接选择 PCL 常见的版本根目录
- 输出目录必须是新的空目录
- 自动建服当前支持：
  - `Fabric`
  - `Forge`
  - `NeoForge`
- `Quilt` 当前只保留模组筛选识别，不进入自动建服流程

## 注意事项

- 模组联网分类依赖网络，速度会受 `Modrinth / MC百科` 请求情况影响
- 一键建服不会修改客户端源目录
- 如果版本清单中无法可靠识别出精确加载器版本，程序会直接报错，不做模糊推断
- 服务端首次初始化后会自动写入：
  - `eula=true`
  - `online-mode=false`
