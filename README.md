# 自动筛选模组分类器 3.01

这个 README 对应当前主线的 `3.01` 版本内容。

相比 `2.01`，这一版不只是补几个功能点，而是把界面、模组筛选、一键开服、下载链路、Java 适配和报错诊断整条流程重新梳理了一遍。目标很直接：少手动折腾，少来回补步骤，让整合包从导入到制作服务端更顺。

如果你想看 `2.01` 的对外发布版本，请查看标签 `v2.0.1` 或切回对应历史提交。

## 这是什么

自动筛选模组分类器主要解决两件事：

1. 帮你把客户端模组快速分类，尽量自动区分哪些能留在服务端，哪些是纯客户端模组，哪些还需要人工确认。
2. 帮你从客户端实例目录、`mrpack` 包或 `zip` 整合包直接制作服务端，尽量把下载依赖、识别版本、准备 Java、首次启动验证这些麻烦步骤串起来。

## 3.01 当前版本重点

1. 前端界面重做，基于新的 Qt 界面结构和主题体系整理了页面层级，整体更清晰，信息密度也更合理。
2. 实时滚动日志、阶段状态和结果统计重新整理过，处理过程比旧版更容易看懂。
3. 支持 `mrpack` 和 `zip` 整合包直接导入开服，下载完包后不需要再手动导入启动器再转一遍。
4. 下载链路更新为智能选源，会按情况优先选择更合适的官方源或镜像源，加快依赖和安装器下载。
5. Java 适配逻辑重新整理过，会更快、更准确地寻找本机可用 Java，找不到时也支持自动下载。
6. 模组自动匹配和分类逻辑重新做过一轮，准确率和速度都有明显提升。实测 300 个模组的筛选耗时大约 2 分钟，速度比旧版快了 3 倍以上。
7. 开服失败时会补充更具体的报错判断，整理关键错误片段，并支持一键复制日志，方便直接发给 DeepSeek 之类的 AI 工具继续排查。
8. 会清理制作服务端过程里留下的一部分缓存和临时内容，减少不必要的磁盘占用。
9. 下载入口和补充链接已经补充，GitHub Release 可以直接下，额外链接见评论区配图。
10. 修复了一批旧版已知问题，包括主题、弹窗、日志反馈、导入链路和下载稳定性这类高频问题。

## 核心功能

### 模组筛选

- 自动分类为 `服务端保留 / 纯客户端 / 待人工确认`
- 支持 `Fabric / Quilt / Forge / NeoForge`
- 优先读取模组自身元数据，再按需要继续查本地离线库、Modrinth、MC百科、CurseForge
- 支持离线库优先命中，减少不必要的联网查询
- 输出 `CSV / JSON / TXT` 报告，并保留更直观的结果预览

### 一键制作服务端

- 支持直接导入客户端实例目录、`mrpack` 整合包、`zip` 整合包
- 自动识别 `Minecraft` 版本、加载器类型和精确加载器版本
- 支持 `Fabric / Forge / NeoForge` 自动制作服务端
- 自动下载安装器、服务端依赖和常用文件
- 自动复制可用模组、常见配置目录和必要启动文件
- 自动查找本机 Java，缺失时支持自动下载
- 自动首启，写入 `eula=true`，并做一次启动验证

### 开服失败辅助排错

- 会按常见问题类型做初步判断，例如依赖缺失、版本不匹配、纯客户端模组混入服务端、Java 版本不对
- 会整理更长的关键报错片段，方便自己看，也方便直接复制出去继续问 AI
- 支持一键复制错误日志

## 运行和打包

源码直接运行：

```powershell
pythonw .\自动筛选模组分类器.pyw
```

或者双击：

```text
启动自动筛选模组分类器.bat
```

重新打包就运行：

```text
打包发布.bat
```

脚本会在 `dist` 目录下生成两个文件：

- `自动筛选模组分类器.exe`
- `auto-mod-classifier-3.01.exe`

第二个文件名是给 GitHub Release 准备的。

## 使用建议

- 如果你本地已经放了离线模组库，建议优先开启本地库查询，速度会更稳。
- 如果整合包是 `mrpack` 或 `zip`，可以直接导入，不需要先走启动器中转。
- 如果首启失败，先看程序整理出的诊断信息和关键报错片段，再决定要不要继续手动排查。

## 需要注意

- `Quilt` 目前保留模组筛选识别，但不进入自动制作服务端流程。
- `MC百科` 和部分 `CurseForge` 兜底链路依赖真实浏览器，机器上最好有 `Chrome` 或 `Edge`。
- 联网筛选速度会受网络环境影响，但当前版本已经尽量把本地判断和多源查询顺序做了优化。
- 一键制作服务端不会修改客户端源目录。
